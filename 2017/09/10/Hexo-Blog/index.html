教你如何用 Github+Hexo 搭一个博客

在搭建博客之前，我们需要做一些准备工作，了解 Git, Github , Github Pages, Hexo 和 Node.js。通俗地说，我们就是用这5样加上一些命令搭成了我们的博客。

Hexo

Hexo 是整个博客的核心部分，它是一个博客系统，基于 Node.js 环境。
Hexo 只需简单的几个命令即可完成博客的生成与部署。Hexo 本地将博客生成静态页面，然后再将其部署至 GitHub。

Node.js

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。在整个搭建过程中起着为 Hexo 系统提供环境的作用。

Git ,Github 和 Github Pages

首先，Git 是一款版本控制系统，Github 是只支持 Git 作为唯一的版本库格式进行托管的托管平台。我们在这里可以简单地理解为 Git 是为 Github 服务的。而 Github 的作用是当作我们网站的服务器，我们需要把我们的所有与博客有关的文件上传到我们个人的 Github 中，上传到的位置就是 Github Pages。
(如果您不了解 Git ,Github，可以去廖雪峰的Git教程和stormzhang的Github教程了解相关知识）
好啦，接下来，就开始我们的搭建博客之旅吧

安装和配置

  1. 安装 Node.js
  2. 在 Github 中创建仓库，名字为username.github.io,将 username换成你自己的 Github 账户名。（注意：必须是这个名字！！！）
  3. 安装Hexo
    win+R 打开控制台，输入

    npm install -g hexo

  4. 初始化
    创建一个文件夹，在控制台中转到这个文件夹，输入：

    hexo init

    例如：

    至此，全部安装工作已经完成

  5. 生成静态页面
    还是在刚刚的目录下，输入：

    hexo g

启动本地服务，进行预览
输入：

hexo server


在浏览器地址栏输入:

http://localhost:4000

看到初始的博客

记得退出调试模式时，按下Ctrl+C

与Github 设置关联

  • 打开控制台或Git Bash，输入如下命令：

    git config –global user.name
    git config –global user.email

(user.name为你的github用户名，user.email为你的github绑定邮箱）

  • 更改_congfig.yml文件
    打开你最初创建的文件夹（即在终端中输入 hexo init命令时所在的位置），找到_config.yml，用记事本打开，下拉到最底端，找到deploy: ,
    更改为：

:后面一定要有空格

  • 安装hexo-deployer-git –save

    npm install hexo-deployer-git –save

  • 将代码部署到github

    hexo deploy

如果你在第6步中没有正确的写好_config.yml文件，此时将出现YAMLException

上传文章

  • 新建文章

    hexo new “article_name”

此时，本地网站目录下的source/_posts文件夹下会出现你新建的文件 article_name.md,打开它并用MarkDown开始编辑，必须使用MarkDown格式才可以看到应有的效果，如果还不曾了解MarkDown，去搜索一下，开始你的MarkDown学习之旅，很简单的！

  • 执行如下三条命令来进行文章的上传

    hexo clean

清除现在的网页

hexo g

重新生成静态页面

hexo deploy

上传到github
以后写文章就是无限循环这几步
